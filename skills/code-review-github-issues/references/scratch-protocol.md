# Scratch Protocol

Use `code-scratch.md` as the only shared memory for the review team.

## Rules

1. Append only.
2. Check coverage before starting a file.
3. Use exact section headers.
4. Keep entries tight.
5. Number issues sequentially.

## Required Sections

- `## STATUS`
- `## QUEUE`
- `## RAW ISSUES`
- `## COVERAGE`
- `## FINAL ISSUE LIST`
- `## GITHUB RESULTS`
- `## LEAD NOTES`

## RAW ISSUE Shape

Use entries with:

- `ISSUE-N`
- `File`
- `Line(s)`
- `Layer`
- `Label`
- `Title`
- `Body`
- `Suggested fix`
- `Blocks`
- `Blocked by`

## FINAL ISSUE LIST

Order the issues by layer and dependency so GitHub issue numbers match the intended repair sequence.
